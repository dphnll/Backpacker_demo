# Backpacker Product Overview

Версия продукта: `1.1.0.0`  
Статус: первая стабильная версия  
Дата: 2026-06-28

## One-liner

Backpacker — мобильный личный планировщик поездки для самостоятельных путешественников, который собирает идеи, брони, места, события, ссылки и расходы в одну управляемую карточную поездку.

## Product Promise

Backpacker помогает планировать поездку без хаоса из заметок, таблиц, чатов и вкладок. Пользователь быстро складывает все элементы поездки в карточки, раскладывает их по дням, видит бюджет, сохраняет запасные идеи и может поделиться планом или сметой.

## Target User

Основной пользователь — самостоятельный путешественник, который планирует поездку сам: ищет билеты, отели, кафе, экскурсии, необычные места, бани, прогулки, запасные варианты и хочет держать все в одном месте.

Типичный сценарий:

- поездка планируется постепенно;
- источников много: сайты, мессенджеры, карты, заметки, рекомендации;
- часть идей еще не решена;
- бюджет важен, но не должен превращать продукт в бухгалтерию;
- телефон — главный рабочий экран во время поездки.

## Core Job To Be Done

Когда я планирую поездку из множества мест, броней, идей и расходов, я хочу быстро собрать все в один живой план по дням, чтобы не терять важное, видеть бюджет и спокойно менять маршрут без переписывания таблиц.

## Current Product Shape

Backpacker V1.1.0.0 состоит из пяти основных зон:

- Главная: тренажер, создание поездки, список поездок, быстрые инфо-действия.
- План по дням: дорожки дней, карточки событий, парковка идей.
- Корзина/идеи: группировка карточек по статусам и запасным вариантам.
- Бюджет: расходы, оплачено, возможный итог, остаток.
- Валюты и экспорт: калькулятор валют, PDF/XLS выгрузки, share-сценарии.

## Key Capabilities

### Trip Basket

Пользователь может создать поездку, задать направление, даты, валюту и бюджет. Поездка хранится локально в браузере и доступна без backend.

### Cards

Карточка хранит:

- название;
- тип;
- статус;
- приоритет;
- дату;
- время;
- длительность;
- цену;
- оплаченную сумму;
- участника/владельца расхода;
- ссылку;
- локацию;
- заметку.

Карточка остается валидной даже без даты, времени или длительности.

### Day Tracks

Дни поездки представлены как дорожки. Карточки можно раскладывать по дням, менять порядок и переносить между днями. Поведение drag/drop, автоскролла и мобильных свайпов считается каноническим для V1.1.0.0.

### Budget

Бюджет показывает:

- общий лимит поездки;
- уже оплачено;
- обязательные траты;
- опциональные траты;
- возможный итог;
- остаток.

Цель бюджета — быстро понять картину поездки, а не заменить полноценный финансовый учет.

### Export

Пользователь может выгрузить:

- смету;
- план по дням;
- PDF;
- XLS/Excel.

В XLS время фиксируется как текст, чтобы Excel не конвертировал его в дату. В смете есть итоговая строка по денежным колонкам.

### Trainer

Тренажер — демо-поездка, в которой можно потрогать карточки, дни, бюджет и экспорт без риска испортить реальную поездку.

## UX Principles

- Mobile-first.
- Карточки важнее таблиц.
- Пустые данные не ломают карточку.
- Drag/drop не должен конфликтовать со скроллом.
- Нижние действия должны быть под рукой.
- Интерфейс должен помогать планировать, а не объяснять себя длинными инструкциями.
- Экспорт должен быть практичным: открыл, отправил, вставил в Excel.

## Stable Baseline

Первая стабильная версия зафиксирована в `BACKPACKER_STABLE_V1_1_0_0.md`.

Особо хрупкие зоны:

- drag/drop карточек;
- автоскролл при перетаскивании;
- свайпы дорожек дней;
- mobile touch balance;
- sticky actions в форме карточки;
- XLS-экспорт времени.

Любая работа в этих зонах требует ручной проверки на desktop и mobile.

## Versioning

Backpacker использует формат:

`super-release.major-epic.minor-feature.patch`

Текущая версия:

`1.1.0.0`

Расшифровка:

- `1` — первая стабильная продуктовая линия.
- `1` — первый большой эпик: личный планировщик поездки.
- `0` — нет отдельной мини-фичи после стабильной базы.
- `0` — нет патча после стабильной базы.

## Non-goals For Current Version

В V1.1.0.0 намеренно нет:

- backend;
- аккаунтов;
- совместного редактирования;
- карт;
- бронирования;
- платежей;
- полноценного multi-device sync.

## Release Checklist

Перед релизным push:

1. Обновить `APP_VERSION`.
2. Обновить `APP_RELEASE_SUMMARY`.
3. Обновить `CACHE_NAME` в `service-worker.js`, если менялись публичные файлы приложения.
4. Прогнать `node --check app.js`.
5. Прогнать `git diff --check`.
6. Для публичного demo пушить в `origin` и `demo`.
